# REQWISE

## Human-First Requirement Validation & QA Performance Assessment Platform

REQWISE is a production-ready Web platform designed to manage, validate, and audit software requirement specifications. Pivoting away from AI-autogenerated reports, REQWISE champions a **Human-First Philosophy**: human QA analysts perform manual audits and author findings, while the platform's rule engines programmatically validate checklists, calculate coverage, check pattern similarity, flag version conflicts, and track QA errors.

---

## 1. System Architecture & Workflows

### 1.1 Requirement State Machine
Requirement documents cycle through the following strict state transitions:
*   `DRAFT`: Created/uploaded by a client.
*   `ASSIGNED`: Admin assigns a QA Analyst.
*   `UNDER_ANALYSIS`: QA opens the spec, reviews category-specific checklists, and marks off items.
*   `REPORT_GENERATED`: QA drafts manual report findings (summary, missing features, risks).
*   `UNDER_REVIEW`: QA submits the final report to the Admin.
*   `CLIENT_REVIEW`: Admin approves the report, forwarding it to the Client.
*   `REVALIDATION`: Admin rejects the report (returns it to the QA for re-work).
*   `FINALIZED`: Client accepts the report, archiving it.

### 1.2 QA Mistake & Assessment Engine
To audit QA human validation capabilities, the system programmatically tracks mistakes and triggers mandatory competency assessments:
*   **Mistake Severities**: `LOW` (1 point), `MEDIUM` (3 points), `HIGH` (5 points).
*   **Assessment Trigger Threshold**: Cumulative weight **$\ge 10$ points**.
*   **Trigger Conditions**:
    1.  *Same Project*: $\ge 2$ instances of the same mistake type in the same project.
    2.  *Same Category*: Same mistake type across $\ge 3$ projects within the same category.
*   **Assessment**: Once triggered, a mandatory assessment widget blocks the QA analyst's dashboard, requiring answers to competency questions about requirements engineering.

## 2. Technology Stack

*   **Frontend**: React (Vite, TypeScript, Tailwind CSS, Zustand state management, React Hook Form, Recharts).
*   **Backend**: Node.js (Express, Mongoose ODM, Zod schema validations, JWT tokens, Bcrypt password hashing).
*   **Database**: MongoDB.

## 3. Setup Guide

### 3.1 Prerequisites
*   Node.js (v18 or higher)
*   MongoDB running locally on port `27017`

### 3.2 Backend Setup
1.  Navigate to the backend directory:
    ```bash
    cd backend
    ```
2.  Install dependencies:
    ```bash
    npm install
    ```
3.  Configure environment variables:
    Create a `.env` file from the template:
    ```bash
    cp .env.example .env
    ```
    Ensure the `MONGODB_URI` points to your active database.
4.  Start the backend development server:
    ```bash
    npm run dev
    ```
    The API will run on `http://localhost:5000`.

### 3.3 Frontend Setup
1.  Navigate to the frontend directory:
    ```bash
    cd ../frontend
    ```
2.  Install dependencies:
    ```bash
    npm install
    ```
3.  Start the frontend development server:
    ```bash
    npm run dev
    ```
    The application will load at `http://localhost:5173`.

## 5. Security & Performance Benchmarks

*   **API Route Protections**: Every sensitive backend endpoint is protected by JWT authorization and RBAC role validation middleware (returning `401`/`403` if unauthorized).
*   **Bundle Size**: Rollup bundle splitting isolates large charting and vendor packages, maintaining main production JS chunks under the **400 KB** limit.
*   **Performance Load**: All key API actions (Login, Dashboard payload compilation, Requirement index, and Analytics calculations) complete within **< 2s**.
